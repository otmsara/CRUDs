Chapter 1 · Foundations

Why a Dedicated Auth Service?

Authentication is one of the most sensitive parts of any system. Isolating it in its own microservice is not over-engineering — it is the correct engineering decision for security, scalability, and team autonomy.

The Core Idea

In a microservices architecture, every service has one job. The Auth Service's job is to answer a single question: who are you, and are you allowed? It handles user identity — registration, credential validation, token issuance, and revocation — so that no other service ever has to touch a password or manage tokens.

Other services (orders, products, notifications) are then stateless with respect to identity: they simply trust the JWT the API Gateway has already validated.

What This Service Covers

POST /auth/register
Registration
Create an account, hash the password, send a verification email.
POST /auth/login
Login
Validate credentials, return access + refresh JWT pair.
POST /auth/refresh
Token Refresh
Exchange a valid refresh token for a new access token.
POST /auth/logout
Logout
Blacklist the current tokens in Redis so they are immediately invalid.
POST /auth/forgot-password
Forgot Password
Generate a secure reset token and send it by email.
POST /auth/reset-password
Reset Password
Validate the reset token, update the password hash.

JWT: Access vs Refresh Tokens

This service issues a two-token strategy, which is the industry standard:

TokenLifetimeStored where?Purpose
access_token15–30 minMemory / Authorization headerProve identity on every API call
refresh_token7–30 daysHttpOnly cookie or secure storageGet a new access token without re-login
Why?
Short-lived access tokens limit the damage if a token is stolen — it expires quickly. Long-lived refresh tokens only travel to the auth service on refresh, reducing their exposure. This is the OAuth2 / OIDC standard pattern.

from sqlalchemy import String, Boolean, DateTime, ForeignKey, func
from sqlalchemy.orm import Mapped, mapped_column, relationship
from datetime import datetime
from app.database import Base

class User(Base):
    __tablename__ = "users"

    id:           Mapped[int]      = mapped_column(primary_key=True)
    email:        Mapped[str]      = mapped_column(String(255), unique=True, index=True)
    hashed_password: Mapped[str]   = mapped_column(String(255))
    is_active:    Mapped[bool]     = mapped_column(Boolean, default=True)
    is_verified:  Mapped[bool]     = mapped_column(Boolean, default=False)
    created_at:   Mapped[datetime]  = mapped_column(DateTime, server_default=func.now())

    refresh_tokens: Mapped[list["RefreshToken"]] = relationship(back_populates="user", cascade="all, delete")
    reset_tokens:   Mapped[list["PasswordResetToken"]] = relationship(back_populates="user", cascade="all, delete")

class RefreshToken(Base):
    # Stores refresh tokens in the DB for rotation + revocation
    __tablename__ = "refresh_tokens"

    id:         Mapped[int]      = mapped_column(primary_key=True)
    user_id:    Mapped[int]      = mapped_column(ForeignKey("users.id"))
    token_hash: Mapped[str]      = mapped_column(String(255), unique=True)
    expires_at: Mapped[datetime]  = mapped_column(DateTime)
    revoked:    Mapped[bool]     = mapped_column(Boolean, default=False)
    created_at: Mapped[datetime]  = mapped_column(DateTime, server_default=func.now())

    user: Mapped["User"] = relationship(back_populates="refresh_tokens")

class PasswordResetToken(Base):
    __tablename__ = "password_reset_tokens"

    id:         Mapped[int]      = mapped_column(primary_key=True)
    user_id:    Mapped[int]      = mapped_column(ForeignKey("users.id"))
    token_hash: Mapped[str]      = mapped_column(String(255), unique=True)
    expires_at: Mapped[datetime]  = mapped_column(DateTime)
    used:       Mapped[bool]     = mapped_column(Boolean, default=False)
    created_at: Mapped[datetime]  = mapped_column(DateTime, server_default=func.now())

    user: Mapped["User"] = relationship(back_populates="reset_tokens")
Note
We store a hash of the refresh token and the reset token in the database, never the raw token itself. If the database is leaked, the attacker still cannot use the tokens. The raw token is only ever sent to the user over HTTPS.

Registration & Password Hashing

Registration is the first touch-point with a user's identity. The golden rule: never store a plain-text password. We use bcrypt, which is intentionally slow — making brute-force attacks computationally expensive.

Password Hashing Utility

Bcrypt stores the salt inside the hash itself. Every time you hash the same password, you get a different result — which also means you cannot compare hashes directly; you must use verify_password.

app/utils/password.py
from passlib.context import CryptContext

# bcrypt with cost factor 12 (2^12 iterations — strong but not painfully slow)
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(plain: str) -> str:
    return pwd_context.hash(plain)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

# Example result of hash_password("mySecret1"):
# $2b$12$eXbDnzpI9AB/EKd7kqTq9.7yB2YPHfHLqz7.XzW8eYiJgbRe1lChy
# ──┬──────────────── salt ──────────────── hash ──────────────
# $2b$12$ → bcrypt with 12 rounds

Registration Service Logic

app/services/auth_service.py (register)
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from fastapi import HTTPException, status
from app.models import User
from app.utils.password import hash_password
from app.schemas import RegisterRequest

async def register_user(data: RegisterRequest, db: AsyncSession) -> User:
    # 1. Check email uniqueness
    result = await db.execute(select(User).where(User.email == data.email))
    existing = result.scalars().first()
    if existing:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="Email already registered"
        )

    # 2. Hash the password — NEVER store plain text
    hashed = hash_password(data.password)

    # 3. Persist the new user
    user = User(email=data.email, hashed_password=hashed)
    db.add(user)
    await db.commit()
    await db.refresh(user)

    return user

Timing
Always run hash_password even when the user doesn't exist (to prevent timing-based user enumeration). An attacker can deduce registered emails if your "wrong email" path returns instantly while "wrong password" takes 100 ms for bcrypt. Return the same generic error for both.

Chapter 5 · Core Auth Flows

Login & JWT Issuance

Login validates credentials and returns a short-lived access token and a long-lived refresh token. Understanding the JWT structure helps you debug and secure your system.

JWT Utility

A JWT has three base64url-encoded parts separated by dots: header (algorithm), payload (claims), and signature. The signature is computed with your JWT_SECRET — any tampering invalidates it.

app/utils/jwt.py
from jose import jwt, JWTError
from datetime import datetime, timedelta, timezone
from app.config import settings

def create_access_token(user_id: int) -> str:
    expire = datetime.now(timezone.utc) + timedelta(
        minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES
    )
    payload = {
        "sub":  str(user_id),   # subject — who owns this token
        "type": "access",         # so we can reject refresh tokens here
        "exp":  expire,
        "iat":  datetime.now(timezone.utc)
    }
    return jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALGORITHM)

def create_refresh_token(user_id: int) -> str:
    expire = datetime.now(timezone.utc) + timedelta(
        days=settings.REFRESH_TOKEN_EXPIRE_DAYS
    )
    payload = {
        "sub":  str(user_id),
        "type": "refresh",        # cannot be used as an access token
        "exp":  expire,
        "iat":  datetime.now(timezone.utc)
    }
    return jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALGORITHM)

def decode_token(token: str) -> dict:
    try:
        payload = jwt.decode(token, settings.JWT_SECRET, algorithms=[settings.JWT_ALGORITHM])
        return payload
    except JWTError:
        return None

Login Service

app/services/auth_service.py (login)
import hashlib
from datetime import datetime, timedelta, timezone
from app.models import RefreshToken
from app.utils.jwt import create_access_token, create_refresh_token
from app.utils.password import verify_password

async def login_user(data: LoginRequest, db: AsyncSession):
    # 1. Look up user by email
    result = await db.execute(select(User).where(User.email == data.email))
    user = result.scalars().first()

    # 2. Verify password — use dummy hash if user not found (timing-safe)
    dummy_hash = "$2b$12$invalidhashtopreventtimingattacks"
    stored_hash = user.hashed_password if user else dummy_hash
    password_ok = verify_password(data.password, stored_hash)

    if not user or not password_ok:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid credentials"  # always the same message!
        )

    # 3. Create token pair
    access_token  = create_access_token(user.id)
    refresh_token = create_refresh_token(user.id)

    # 4. Persist a HASH of the refresh token in the DB
    token_hash = hashlib.sha256(refresh_token.encode()).hexdigest()
    rt = RefreshToken(
        user_id    = user.id,
        token_hash = token_hash,
        expires_at = datetime.now(timezone.utc) + timedelta(days=7)
    )
    db.add(rt)
    await db.commit()

    return {
        "access_token":  access_token,
        "refresh_token": refresh_token,
        "token_type":    "bearer"
    }

Tip
The access token payload is not encrypted — it's base64-encoded and signed. Anyone who holds it can decode the claims (user id, expiry). Never put sensitive data (SSN, password, credit card) in a JWT payload. Only store non-sensitive identifiers.

Chapter 6 · Core Auth Flows

Token Refresh Strategy

When an access token expires, the client silently sends its refresh token to get a new pair. This keeps users logged in without re-entering their password — while maintaining short-window security.

The Refresh Flow

1
Client receives 401 on an API call
The access token has expired (exp is in the past).
2
Client calls POST /auth/refresh with the refresh token
Sends {"refresh_token": "eyJ..."} in the request body.
3
Server verifies the refresh token
Decodes JWT, checks type == "refresh", looks up the hash in DB, ensures it's not revoked or expired.
4
Server rotates: revokes old, issues new pair
Old DB record is marked revoked = True. New access + refresh tokens are returned.

Refresh Service

app/services/auth_service.py (refresh)
async def refresh_tokens(refresh_token: str, db: AsyncSession):
    # 1. Decode and validate JWT structure
    payload = decode_token(refresh_token)
    if not payload or payload.get("type") != "refresh":
        raise HTTPException(401, "Invalid refresh token")

    user_id = int(payload["sub"])

    # 2. Find matching DB record by hash
    token_hash = hashlib.sha256(refresh_token.encode()).hexdigest()
    result = await db.execute(
        select(RefreshToken)
        .where(RefreshToken.token_hash == token_hash)
        .where(RefreshToken.revoked == False)
    )
    db_token = result.scalars().first()
    if not db_token:
        raise HTTPException(401, "Refresh token revoked or not found")

    # 3. Revoke the old token (token rotation)
    db_token.revoked = True

    # 4. Issue a new pair
    new_access  = create_access_token(user_id)
    new_refresh = create_refresh_token(user_id)

    new_hash = hashlib.sha256(new_refresh.encode()).hexdigest()
    db.add(RefreshToken(
        user_id    = user_id,
        token_hash = new_hash,
        expires_at = datetime.now(timezone.utc) + timedelta(days=7)
    ))
    await db.commit()

    return {"access_token": new_access, "refresh_token": new_refresh, "token_type": "bearer"}

Refresh Endpoint

app/routers/auth.py (refresh)

Refresh token reuse detection: if you see a revoked refresh token being submitted, it means a token theft occurred and both the old and new tokens should be invalidated. Mark all refresh tokens for that user as revoked and force re-login.

Chapter 7 · Core Auth Flows

Logout & Token Blacklist

JWTs are stateless — they're valid until they expire. Logout requires actively invalidating them. We use Redis as a fast blacklist: the token's JTI (or hash) is stored with a TTL matching the token's remaining lifetime.

The Problem with Stateless JWTs

You cannot "delete" a JWT the way you delete a session ID from the database. Once issued, a JWT is self-contained and valid until exp. So a user who logs out but whose token was stolen still has a valid token in the attacker's hands.

The solution: a Redis blacklist. On logout, we store the token's unique identifier in Redis with a TTL equal to the remaining lifetime. Every protected endpoint checks this blacklist before trusting a token.

Redis Utility

app/utils/redis.py
import redis.asyncio as aioredis
from app.config import settings

redis_client: aioredis.Redis = None

async def get_redis() -> aioredis.Redis:
    global redis_client
    if redis_client is None:
        redis_client = await aioredis.from_url(settings.REDIS_URL)
    return redis_client

async def blacklist_token(jti: str, ttl_seconds: int):
    """Store a token ID in Redis so it's rejected on every future request."""
    r = await get_redis()
    await r.setex(f"blacklist:{jti}", ttl_seconds, "1")

async def is_blacklisted(jti: str) -> bool:
    r = await get_redis()
    return await r.exists(f"blacklist:{jti}") == 1

Updating the JWT to include a JTI

We add a jti (JWT ID) claim — a unique random string — to every token. This is what we blacklist, not the full token string.

app/utils/jwt.py (updated with jti)
import uuid

def create_access_token(user_id: int) -> str:
    expire = datetime.now(timezone.utc) + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    payload = {
        "sub":  str(user_id),
        "type": "access",
        "jti":  str(uuid.uuid4()),   # unique identifier for THIS token
        "exp":  expire,
        "iat":  datetime.now(timezone.utc)
    }
    return jwt.encode(payload, settings.JWT_SECRET, algorithm=settings.JWT_ALGORITHM)

Protected Route Dependency

Every secured endpoint uses this dependency to authenticate the request.

app/utils/jwt.py (get_current_user dependency)
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

bearer_scheme = HTTPBearer()

async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(bearer_scheme),
    db: AsyncSession = Depends(get_db)
) -> User:
    token = credentials.credentials

    # 1. Decode and validate JWT
    payload = decode_token(token)
    if not payload or payload.get("type") != "access":
        raise HTTPException(401, "Invalid access token")

    # 2. Check blacklist
    jti = payload.get("jti")
    if jti and await is_blacklisted(jti):
        raise HTTPException(401, "Token has been revoked")

    # 3. Load user from DB
    user_id = int(payload["sub"])
    result = await db.execute(select(User).where(User.id == user_id))
    user = result.scalars().first()
    if not user or not user.is_active:
        raise HTTPException(401, "User not found or deactivated")

    return user

Logout Endpoint

app/routers/auth.py (logout)
from app.utils.redis import blacklist_token
from datetime import timezone

@router.post("/logout", status_code=204)
async def logout(
    credentials: HTTPAuthorizationCredentials = Depends(bearer_scheme),
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user)
):
    """Blacklist the current access token and revoke all refresh tokens."""
    token = credentials.credentials
    payload = decode_token(token)

    # Blacklist the access token in Redis until it would expire naturally
    jti = payload.get("jti")
    exp = payload.get("exp")
    now = datetime.now(timezone.utc).timestamp()
    ttl = int(exp - now)
    if ttl > 0:
        await blacklist_token(jti, ttl)

    # Revoke ALL refresh tokens for this user in DB
    await db.execute(
        update(RefreshToken)
        .where(RefreshToken.user_id == current_user.id)
        .where(RefreshToken.revoked == False)
        .values(revoked=True)
    )
    await db.commit()
    # 204 No Content — no body returned

Chapter 8 · Password Recovery

Forgot Password Flow

The forgot-password endpoint sends a time-limited, single-use reset link to the user's email. The link contains a cryptographically random token — not the user's ID, not a JWT.

Why Not a JWT for Reset?

JWTs are self-verifying — you cannot "use them once" without storing state. A random token stored in the database (as a hash) is the correct choice: it can be verified, marked as used, and expired independently.

The Full Flow

1
User submits POST /auth/forgot-password with their email
Always return 200 OK with the same message — even if the email isn't registered. This prevents email enumeration.
2
Server generates a secure random token
secrets.token_urlsafe(32) — 32 bytes = 256 bits of entropy. Cannot be guessed.
3
Token hash is stored in the DB; raw token is sent by email
The DB stores sha256(token). Only the user receives the raw token via email.
4
Email contains a link: frontend_url/reset-password?token=…
The frontend reads the token from the URL and sends it to POST /auth/reset-password.

Forgot Password Service

app/services/auth_service.py (forgot_password)
import secrets, hashlib
from datetime import datetime, timedelta, timezone
from app.models import PasswordResetToken
from app.services.email_service import send_reset_email
from app.config import settings

RESET_TOKEN_EXPIRE_MINUTES = 30

async def forgot_password(email: str, db: AsyncSession):
    # Always return success — never reveal whether email exists
    result = await db.execute(select(User).where(User.email == email))
    user = result.scalars().first()

    if user:  # only act if user exists, but don't tell the caller
        # 1. Expire any previous unused reset tokens for this user
        await db.execute(
            update(PasswordResetToken)
            .where(PasswordResetToken.user_id == user.id)
            .where(PasswordResetToken.used == False)
            .values(used=True)
        )

        # 2. Generate the raw token (sent to user) and its hash (stored in DB)
        raw_token  = secrets.token_urlsafe(32)
        token_hash = hashlib.sha256(raw_token.encode()).hexdigest()
        expires_at = datetime.now(timezone.utc) + timedelta(minutes=RESET_TOKEN_EXPIRE_MINUTES)

        # 3. Persist the hashed token
        db.add(PasswordResetToken(
            user_id    = user.id,
            token_hash = token_hash,
            expires_at = expires_at
        ))
        await db.commit()

        # 4. Send email with the raw token embedded in the reset link
        reset_link = f"{settings.FRONTEND_URL}/reset-password?token={raw_token}"
        await send_reset_email(email=user.email, reset_link=reset_link)

    # Return the same response regardless
    return {"message": "If this email is registered, you will receive a reset link."}

Reset Password Endpoint

The reset endpoint receives the raw token from the URL, validates it against the database, updates the password, and immediately invalidates the token so it can only be used once.

Reset Password Service

app/services/auth_service.py (reset_password)
async def reset_password(token: str, new_password: str, db: AsyncSession):
    # 1. Hash the incoming token to look it up in DB
    token_hash = hashlib.sha256(token.encode()).hexdigest()

    # 2. Find a valid, unused, non-expired record
    now = datetime.now(timezone.utc)
    result = await db.execute(
        select(PasswordResetToken)
        .where(PasswordResetToken.token_hash == token_hash)
        .where(PasswordResetToken.used       == False)
        .where(PasswordResetToken.expires_at  > now)
    )
    db_token = result.scalars().first()

    if not db_token:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="Reset token is invalid or has expired"
        )

    # 3. Load the user
    result = await db.execute(select(User).where(User.id == db_token.user_id))
    user = result.scalars().first()

    # 4. Update the password
    user.hashed_password = hash_password(new_password)

    # 5. Mark the token as used — single use!
    db_token.used = True

    # 6. Revoke all existing refresh tokens (force re-login everywhere)
    await db.execute(
        update(RefreshToken)
        .where(RefreshToken.user_id == user.id)
        .values(revoked=True)
    )

    await db.commit()
    return {"message": "Password updated successfully"}

Security Best Practices

An auth service is the highest-value target in your system. These hardening measures are not optional — they are the difference between a secure service and a liability.

Rate Limiting on Sensitive Endpoints

Brute-force attacks on login, and spam on forgot-password, are trivially easy without rate limiting. Use Redis to count attempts per IP or per email.

app/utils/rate_limit.py
from fastapi import HTTPException, Request
from app.utils.redis import get_redis

async def rate_limit(key: str, max_attempts: int, window_seconds: int):
    """Increment a counter in Redis. Raise 429 if max_attempts exceeded."""
    r = await get_redis()
    count = await r.incr(key)
    if count == 1:
        await r.expire(key, window_seconds)  # start the window on first hit
    if count > max_attempts:
        raise HTTPException(
            status_code=429,
            detail=f"Too many attempts. Try again in {window_seconds} seconds."
        )

How the API Gateway Integrates

The gateway validates every incoming JWT using the shared JWT_SECRET. The auth service never needs to be called on every request — only on login, refresh, and logout.

api-gateway: JWT validation middleware
# In the API Gateway, import the same decode_token utility
from shared.jwt import decode_token

@app.middleware("http")
async def auth_middleware(request: Request, call_next):
    PUBLIC_PATHS = ["/auth/login", "/auth/register", "/auth/refresh",
                    "/auth/forgot-password", "/auth/reset-password", "/health"]

    if request.url.path in PUBLIC_PATHS:
        return await call_next(request)  # no auth needed

    token = request.headers.get("Authorization", "").removeprefix("Bearer ")
    payload = decode_token(token)
    if not payload:
        return JSONResponse({"detail": "Unauthorized"}, status_code=401)

    # Inject user ID so downstream services can use it without re-verifying
    request.state.user_id = payload["sub"]
    response = await call_next(request)
    return response
