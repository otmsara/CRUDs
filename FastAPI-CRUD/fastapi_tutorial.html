Chapter 01

Web App Architecture & Theory

Understand the client-server model and how FastAPI fits into a modern REST API architecture before writing a single line of code.

The Client-Server Model

A web application consists of two main parts: a client (browser, mobile app, or another service) and a server. The client sends HTTP requests; the server processes them and returns responses.

💡
FastAPI is the server. It receives HTTP requests, runs your Python logic, talks to a database, and returns JSON responses to whoever called it.

REST API Principles

  • Resources — everything is a resource (employees, users, departments)
  • HTTP Methods — GET (read), POST (create), PUT/PATCH (update), DELETE (remove)
  • Stateless — each request carries all needed information
  • JSON — standard data exchange format

Our Project Architecture

1
Client → HTTP Request
A frontend, Postman, or curl sends a request like GET /employees/42
2
FastAPI Router
FastAPI matches the URL to a Python function (called a path operation)
3
Business Logic
Your function validates input (via Pydantic), checks auth (JWT), runs logic
4
Database (SQLAlchemy)
Data is fetched or written to SQLite via SQLAlchemy ORM
5
JSON Response
FastAPI serializes the result to JSON and returns it with proper HTTP status code

FastAPI Setup & Basics

Create your first FastAPI application and understand the core concepts: the app instance, decorators, and path operations.

Your First FastAPI App

pythonmain.py
from fastapi import FastAPI

# 1. Create the FastAPI instance — this IS your application
app = FastAPI(
    title="Employees API",
    description="Manage company employees",
    version="1.0.0"
)

# 2. A path operation — GET request at the root URL "/"
@app.get("/")
def root():
    return {"message": "Employees API is running 🚀"}

# 3. Another route example
@app.get("/health")
def health_check():
    return {"status": "ok"}

Run the Server

bash
# --reload = auto-restart on file changes (dev only)
uvicorn main:app --reload

# Output:
# INFO:     Uvicorn running on http://127.0.0.1:8000
💡
Anatomy of @app.get("/"): app is your FastAPI instance · .get is the HTTP method · "/" is the URL path. The function below it runs when that URL is requested.

Key Concepts

  • Path operation decorator — @app.get(), @app.post(), etc. register routes
  • Path operation function — the Python function that runs when the route is hit
  • Automatic JSON — return a dict and FastAPI auto-converts it to JSON
  • Async support — use async def instead of def for non-blocking I/O

Chapter 06

Path Parameters

Capture dynamic values from the URL — like an employee ID — using Python's type annotations.

Defining a Path Parameter

pythonmain.py
# {employee_id} in the path becomes a parameter
@app.get("/employees/{employee_id}")
def get_employee(employee_id: int):
    # FastAPI automatically:
    # 1. Extracts the value from the URL
    # 2. Converts it to int (type annotation)
    # 3. Validates it (errors if not a number)
    
    employee = next(
        (e for e in fake_employees if e["id"] == employee_id),
        None
    )
    return employee
💡
The type annotation employee_id: int does double duty — it tells FastAPI the expected type AND Python type checkers about it. If you pass /employees/abc, FastAPI returns a 422 Unprocessable Entity automatically.

Multiple Path Parameters

python
@app.get("/departments/{dept_id}/employees/{emp_id}")
def get_dept_employee(dept_id: int, emp_id: int):
    return {"dept": dept_id, "employee": emp_id}

Path Parameter Validation

python
from fastapi import Path

@app.get("/employees/{employee_id}")
def get_employee(
    employee_id: int = Path(gt=0, description="Must be positive integer")
):
    return {"id": employee_id}

Raising Errors & Status Codes

Return meaningful HTTP error responses when something goes wrong — like when an employee doesn't exist.

HTTPException

Chapter 08

Query Parameters

Accept optional filtering and search parameters via the URL query string — like ?role=Engineer&limit=10.

Basic Query Parameters

pythonmain.py
# Any function parameter NOT in the URL path = query parameter
@app.get("/employees")
def get_employees(
    skip: int = 0,          # ?skip=10  (default: 0)
    limit: int = 10,        # ?limit=5  (default: 10)
    role: str | None = None  # ?role=Engineer  (optional)
):
    results = fake_employees[skip : skip + limit]
    
    if role:
        results = [e for e in results if e["role"] == role]
    
    return results

How to Call These Endpoints

bash
# Get first 5 employees
GET /employees?limit=5

# Skip first 10, get next 5
GET /employees?skip=10&limit=5

# Filter by role
GET /employees?role=Engineer

# Combine everything
GET /employees?skip=0&limit=20&role=Manager

Query Parameter Validation

python
from fastapi import Query

@app.get("/employees")
def get_employees(
    limit: int = Query(default=10, ge=1, le=100),
    role: str | None = Query(None, min_length=3)
):
    return fake_employees[:limit]
💡
ge=1 means "greater than or equal to 1" · le=100 means "less than or equal to 100". FastAPI validates these and returns 422 automatically if violated.

Request Body & POST

Accept structured JSON data from the client to create new employee records using Pydantic models.

Define the Input Schema

pythonschemas.py
from pydantic import BaseModel, EmailStr, Field
from typing import Optional

# Pydantic model = the shape of data we EXPECT from the client
class EmployeeCreate(BaseModel):
    name: str = Field(min_length=2, max_length=100)
    email: EmailStr
    role: str
    salary: float = Field(gt=0)
    department: Optional[str] = None

# Example valid JSON body:
# {
#   "name": "Alice Johnson",
#   "email": "alice@company.com",
#   "role": "Engineer",
#   "salary": 75000.00
# }

Use It in a POST Route

pythonmain.py
from fastapi import FastAPI, status
from schemas import EmployeeCreate

@app.post("/employees", status_code=status.HTTP_201_CREATED)
def create_employee(employee: EmployeeCreate):
    # FastAPI automatically:
    # 1. Reads the JSON request body
    # 2. Validates it against EmployeeCreate
    # 3. Returns 422 if validation fails
    # 4. Passes a validated object to your function
    
    new_employee = employee.model_dump()
    new_employee["id"] = len(fake_employees) + 1
    fake_employees.append(new_employee)
    
    return new_employee
⚠️
In this example data is stored in memory only. Once the server restarts, it's gone. That's why we add a real database in the next chapters.

Output Type & Pydantic Models

Control exactly what data is returned by defining output schemas — hide sensitive fields like passwords automatically.

Input vs Output Schemas

pythonschemas.py
from pydantic import BaseModel, EmailStr
from datetime import datetime

# INPUT: What the client sends (includes password)
class EmployeeCreate(BaseModel):
    name: str
    email: EmailStr
    role: str
    salary: float
    password: str   # Accept password in creation

# OUTPUT: What we return (NEVER return the password!)
class EmployeeOut(BaseModel):
    id: int
    name: str
    email: EmailStr
    role: str
    salary: float
    created_at: datetime
    
    # Needed in Pydantic v2 to work with SQLAlchemy models
    model_config = {"from_attributes": True}

Database Connection

Connect FastAPI to a SQLite database using SQLAlchemy ORM and set up dependency injection for sessions.

Database Setup

pythondatabase.py
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

# SQLite URL — the file will be created automatically
SQLALCHEMY_DATABASE_URL = "sqlite:///./employees.db"
# For PostgreSQL: "postgresql://user:pass@localhost/dbname"

# Engine = the connection to the database
engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False}  # SQLite only
)

# SessionLocal = a factory for database sessions
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

# Base class for all SQLAlchemy models
Base = declarative_base()

# Dependency: yields a DB session per request, always closes it
def get_db():
    db = SessionLocal()
    try:
        yield db       # <-- request code runs here
    finally:
        db.close()    # <-- always closes, even on errors

SQLAlchemy Model

pythonmodels.py
from sqlalchemy import Column, Integer, String, Float, DateTime
from sqlalchemy.sql import func

Chapter 12

Creating Employees & Saving to Database

Wire the POST endpoint to the database using dependency injection and the SQLAlchemy session.

The Full POST Endpoint

pythonrouters/employees.py
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy.orm import Session
from passlib.context import CryptContext

from database import get_db
import models, schemas

router = APIRouter(prefix="/employees", tags=["Employees"])
pwd_context = CryptContext(schemes=["bcrypt"])

@router.post("", response_model=schemas.EmployeeOut, status_code=status.HTTP_201_CREATED)
def create_employee(
    employee: schemas.EmployeeCreate,
    db: Session = Depends(get_db)  # ← FastAPI injects this!
):
    # Check if email already registered
    existing = db.query(models.Employee).filter(
        models.Employee.email == employee.email
    ).first()
    if existing:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="Email already registered"
        )
    
    # Hash the password before storing
    hashed = pwd_context.hash(employee.password)
    
    # Create SQLAlchemy model instance
    new_emp = models.Employee(
        name=employee.name,
        email=employee.email,
        role=employee.role,
        salary=employee.salary,
        hashed_pw=hashed
    )
    
    db.add(new_emp)      # Stage the insert
    db.commit()          # Write to DB
    db.refresh(new_emp)  # Load the auto-generated ID
    
    return new_emp
ℹ️
Depends(get_db) is dependency injection. FastAPI calls get_db() for each request, passes the session to your function, and handles cleanup automatically.

Register the Router in main.py

Deleting Employees

Implement the DELETE endpoint to remove employees from the database permanently.

DELETE Endpoint

pythonrouters/employees.py
from fastapi import Response

@router.delete("/{employee_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_employee(employee_id: int, db: Session = Depends(get_db)):
    # 1. Find the employee
    employee = db.query(models.Employee).filter(
        models.Employee.id == employee_id
    ).first()
    
    # 2. Raise 404 if not found
    if not employee:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Employee {employee_id} not found"
        )
    
    # 3. Delete and commit
    db.delete(employee)
    db.commit()
    
    # 4. 204 No Content = success, no body returned
    return Response(status_code=status.HTTP_204_NO_CONTENT)
⚠️
Always protect DELETE endpoints with authentication (covered next). Without protection, anyone can delete any employee!

User Authentication & JWT Tokens

Build a login endpoint that verifies credentials and issues a signed JWT access token.

Token Config & Utilities

pythonrouters/auth.py
from fastapi import APIRouter, Depends, HTTPException, status from fastapi.security import OAuth2PasswordRequestForm from jose import jwt from passlib.context import CryptContext from datetime import datetime, timedelta from sqlalchemy.orm import Session from database import get_db import models router = APIRouter(prefix="/auth", tags=["Authentication"]) pwd_context = CryptContext(schemes=["bcrypt"]) # 🔐 Keep this SECRET and in environment variables in production! SECRET_KEY = "your-super-secret-key-change-in-production" ALGORITHM = "HS256" TOKEN_EXPIRE_MINUTES = 30 def create_access_token(data: dict): payload = data.copy() expire = datetime.utcnow() + timedelta(minutes=TOKEN_EXPIRE_MINUTES) payload.update({"exp": expire}) return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM) # Login endpoint @router.post("/login") def login( form: OAuth2PasswordRequestForm = Depends(), db: Session = Depends(get_db) ): # Find user by email (username field used for email) user = db.query(models.Employee).filter( models.Employee.email == form.username ).first() # Verify password if not user or not pwd_context.verify(form.password, user.hashed_pw): raise HTTPException( status_code=status.HTTP_401_UNAUTHORIZED, detail="Invalid credentials", headers={"WWW-Authenticate": "Bearer"} ) # Issue JWT token token = create_access_token({"sub": user.email, "id": user.id}) return {"access_token": token, "token_type": "bearer"}
ℹ️
JWT Structure: A token is 3 base64 parts: header.payload.signature. The payload contains the user's ID/email. The signature verifies it wasn't tampered with.

Chapter 16

Protecting Endpoints

Verify the JWT token on every protected request using a reusable dependency function.

Current User Dependency

pythonrouters/auth.py
from fastapi.security import OAuth2PasswordBearer
from jose import jwt, JWTError

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="auth/login")

def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db)
):
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"}
    )
    
    try:
        # Decode and verify the JWT
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        email: str = payload.get("sub")
        
        if email is None:
            raise credentials_exception
            
    except JWTError:
        raise credentials_exception
    
    # Look up the user in DB
    user = db.query(models.Employee).filter(
        models.Employee.email == email
    ).first()
    
    if user is None:
        raise credentials_exception
    
    return user

Apply to Protected Routes

pythonrouters/employees.py
from routers.auth import get_current_user

# 🔒 Only authenticated users can delete employees
@router.delete("/{employee_id}", status_code=204)
def delete_employee(
    employee_id: int,
    db: Session = Depends(get_db),
    current_user: models.Employee = Depends(get_current_user)  # 🔒
):
    if current_user.id != employee_id:
        raise HTTPException(status_code=403, detail="Not authorized")
    
    employee = db.query(models.Employee).filter(
        models.Employee.id == employee_id
    ).first()
    db.delete(employee)
    db.commit()
