Chapter 01 · Foundations

What Are Microservices?

Microservices is an architectural style where a large application is decomposed into small, independently deployable services — each owning its own data and communicating over the network.

Monolith vs Microservices

In a monolith, all features — users, products, orders, emails — live in one codebase, share one database, and deploy as one unit. When it grows, it becomes hard to scale individual features, hard to deploy safely, and hard for multiple teams to work in parallel.

In a microservices architecture, each feature becomes its own independent service with its own database, its own deployment lifecycle, and its own API.

Note
Microservices add operational complexity. They're the right choice when your team and traffic have outgrown what a monolith can handle — not for small projects.

The 6 Services We'll Build

:8000
api-gateway
Single entry point. Routes requests to downstream services.
:8001
user-service
Registration, login, JWT issuance, user profiles.
:8002
product-service
Product catalog, inventory, pricing.
:8003

Order creation, status tracking, payment events.
:8004

Email/SMS via RabbitMQ events. No HTTP endpoints.
:6379
redis
Shared cache, rate limiting, session storage.

Core Principles We'll Follow

  • Single Responsibility — each service does one thing well
  • Database per Service — services never share a database directly
  • API-first Communication — services talk via HTTP or message queues
  • Independent Deployment — any service can be updated without touching others
  • Failure Isolation — one service crashing doesn't bring down the system

Chapter 02 · Foundations

System Architecture

Map out the full system topology — how services communicate, what protocols they use, and how data flows from a client request to a database write.

Request Flow

1
Client → API Gateway :8000
Every request enters through the gateway. No client talks directly to a service.
2
Gateway validates JWT
Authenticated routes: gateway verifies the token before proxying. Saves each service from re-implementing auth.
3
Gateway proxies → downstream service
Uses httpx to forward the request, injecting user context in a header.
4
Service executes & publishes events
e.g. Order Service creates an order in its DB, then publishes order.created to RabbitMQ.
5
Notification Service consumes event
Asynchronously reads the event and sends an email. The Order Service doesn't wait for this.

Communication Patterns

PatternProtocolUse case
SynchronousHTTP (httpx)Gateway → Services, Service → Service reads
AsynchronousAMQP (RabbitMQ)Order events, notifications, analytics
CacheRedisSession data, rate limiting, hot reads

Database Isolation

Each service gets its own PostgreSQL database. They never query each other's DB. If Order Service needs a product's price, it calls the Product Service API — never reads the product table directly.

Tip
For development we run all Postgres instances in Docker with different port mappings. In production each service would typically have its own managed database cluster.

Shared Libraries

Extract common code — JWT utilities, base schemas, error handlers — into a shared package so every service stays DRY.

shared/auth.py — JWT Utilities

shared/auth.py
from jose import jwt, JWTError
from fastapi import HTTPException, Header
from datetime import datetime, timedelta
import os

SECRET_KEY = os.getenv("JWT_SECRET", "change-in-production")
ALGORITHM = "HS256"

def create_token(data: dict, expire_minutes: int = 60) -> str:
    payload = data | {"exp": datetime.utcnow() + timedelta(minutes=expire_minutes)}
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def decode_token(token: str) -> dict:
    try:
        return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
    except JWTError:
        raise HTTPException(401, "Invalid or expired token")

# Dependency: extract user context from X-User-ID header
# (set by the gateway after verifying the JWT)
def get_user_id(x_user_id: str = Header(None)) -> int:
    if not x_user_id:
        raise HTTPException(401, "Missing user context")
    return int(x_user_id)

User Service

The User Service owns all identity: registration, login, password hashing, and JWT token issuance. It's the only service that writes to the users database.

Endpoints

POST/users/registerCreate account, hash password
POST/users/loginVerify credentials, return JWT
GET/users/meReturn current user profile
GET/users/{user_id}Internal: called by other services

main.py

user-service/main.py
from fastapi import FastAPI, Depends, HTTPException, status
from sqlalchemy.orm import Session
from passlib.context import CryptContext
from database import get_db
import models, schemas
from shared.auth import create_token, get_user_id

app = FastAPI(title="User Service")
pwd = CryptContext(schemes=["bcrypt"])

models.Base.metadata.create_all(bind=engine)

@app.post("/users/register", response_model=schemas.UserOut, status_code=201)
def register(body: schemas.UserCreate, db: Session = Depends(get_db)):
    if db.query(models.User).filter_by(email=body.email).first():
        raise HTTPException(400, "Email already registered")
    user = models.User(
        email=body.email,
        name=body.name,
        hashed_password=pwd.hash(body.password)
    )
    db.add(user); db.commit(); db.refresh(user)
    return user

@app.post("/users/login")
def login(body: schemas.UserLogin, db: Session = Depends(get_db)):
    user = db.query(models.User).filter_by(email=body.email).first()
    if not user or not pwd.verify(body.password, user.hashed_password):
        raise HTTPException(401, "Invalid credentials")
    token = create_token({"sub": user.email, "user_id": user.id})
    return {"access_token": token, "token_type": "bearer"}

@app.get("/users/me", response_model=schemas.UserOut)
def get_me(user_id: int = Depends(get_user_id), db: Session = Depends(get_db)):
    return db.query(models.User).get(user_id)

Product Service

The Product Service manages the catalog. It exposes read endpoints publicly and write endpoints only to authenticated admins. Inventory decrements are triggered by order events.

Endpoints

GET/productsList products (public, cached)
GET/products/{id}Get product (public)
POST/productsCreate product (admin only)
PUT/products/{id}/inventoryUpdate stock (internal only)

main.py

product-service/main.py
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.orm import Session
from database import get_db
import models, schemas

app = FastAPI(title="Product Service")

@app.get("/products", response_model=list[schemas.ProductOut])
def list_products(
    skip: int = 0, limit: int = 20,
    db: Session = Depends(get_db)
):
    return db.query(models.Product).offset(skip).limit(limit).all()

@app.get("/products/{product_id}", response_model=schemas.ProductOut)
def get_product(product_id: int, db: Session = Depends(get_db)):
    p = db.get(models.Product, product_id)
    if not p:
        raise HTTPException(404, "Product not found")
    return p

@app.put("/products/{product_id}/inventory")
def decrement_inventory(
    product_id: int,
    body: schemas.InventoryUpdate,
    db: Session = Depends(get_db)
):
    # Called internally by order-service via HTTP
    p = db.get(models.Product, product_id)
    if p.stock < body.quantity:
        raise HTTPException(409, "Insufficient stock")
    p.stock -= body.quantity
    db.commit()
    return {"remaining_stock": p.stock}

Chapter 07 · Core Services

Order Service

The most complex service — it creates orders, calls Product Service to decrement stock, and publishes an order.created event to RabbitMQ for the Notification Service to consume.

Endpoints

POST/ordersCreate order (auth required)
GET/ordersMy orders (auth required)
GET/orders/{id}Order detail

main.py — Full create_order flow

order-service/main.py
import httpx, json, aio_pika, os
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.orm import Session
from database import get_db
from shared.auth import get_user_id
import models, schemas

app = FastAPI(title="Order Service")
PRODUCT_URL = os.getenv("PRODUCT_SERVICE_URL")
RABBITMQ_URL = os.getenv("RABBITMQ_URL")

@app.post("/orders", response_model=schemas.OrderOut, status_code=201)
async def create_order(
    body: schemas.OrderCreate,
    user_id: int = Depends(get_user_id),
    db: Session = Depends(get_db)
):
    # 1. Verify product exists and has stock
    async with httpx.AsyncClient() as client:
        res = await client.put(
            f"{PRODUCT_URL}/products/{body.product_id}/inventory",
            json={"quantity": body.quantity}
        )
        if res.status_code != 200:
            raise HTTPException(res.status_code, res.json().get("detail"))

    # 2. Persist the order
    order = models.Order(
        user_id=user_id,
        product_id=body.product_id,
        quantity=body.quantity,
        total_price=res.json()["unit_price"] * body.quantity,
        status="CONFIRMED"
    )
    db.add(order); db.commit(); db.refresh(order)

    # 3. Publish event to RabbitMQ (fire-and-forget)
    await publish_event("order.created", {
        "order_id": order.id,
        "user_id": user_id,
        "product_id": body.product_id,
        "total_price": order.total_price
    })
    return order

async def publish_event(routing_key: str, payload: dict):
    conn = await aio_pika.connect_robust(RABBITMQ_URL)
    async with conn:
        channel = await conn.channel()
        exchange = await channel.declare_exchange("orders", aio_pika.ExchangeType.TOPIC)
        await exchange.publish(
            aio_pika.Message(json.dumps(payload).encode()),
            routing_key=routing_key
        )
Watch out
This is a simplified example. In production, use a saga pattern or transactional outbox to avoid partial failures between the DB write and the queue publish.

Notification Service

A pure event consumer — no HTTP server, no database. It listens to the RabbitMQ orders exchange and sends emails when orders are confirmed.

consumer.py — RabbitMQ Listener

notification-service/consumer.py
import asyncio, aio_pika, json, os, smtplib
from email.mime.text import MIMEText

RABBITMQ_URL = os.getenv("RABBITMQ_URL")

async def handle_order_created(message: aio_pika.IncomingMessage):
    async with message.process():     # auto-ack on success
        data = json.loads(message.body)
        print(f"📧 Sending confirmation for order #{data['order_id']}")
        send_email(
            to=f"user_{data['user_id']}@example.com",
            subject=f"Order #{data['order_id']} Confirmed",
            body=f"Your order total: ${data['total_price']:.2f}"
        )

def send_email(to: str, subject: str, body: str):
    # Use smtplib, sendgrid, etc.
    print(f"[MAIL] To: {to} | {subject}")

async def main():
    conn = await aio_pika.connect_robust(RABBITMQ_URL)
    channel = await conn.channel()
    
    exchange = await channel.declare_exchange(
        "orders", aio_pika.ExchangeType.TOPIC, durable=True
    )
    queue = await channel.declare_queue("notification_queue", durable=True)
    await queue.bind(exchange, routing_key="order.*")  # all order events
    
    print("🐇 Notification Service waiting for messages...")
    await queue.consume(handle_order_created)
    await asyncio.Future()  # run forever

asyncio.run(main())
Pattern
Notice this service has no FastAPI app. It's just a plain async Python script. This is perfectly valid — not every microservice needs to be an HTTP server.

Chapter 09 · Gateway & Routing

API Gateway Pattern

The gateway is the single entry point for all clients. It handles authentication, request routing, rate limiting, and response aggregation — shielding services from direct exposure.

Gateway Responsibilities

  • JWT verification — validates the token once, injects user context into forwarded requests
  • Request proxying — forwards requests to the correct service using httpx
  • Rate limiting — per-IP or per-user request throttling via Redis
  • CORS headers — centralised cross-origin policy, services don't need to know about it
  • Request/response logging — one place to log all traffic

gateway/main.py skeleton

api-gateway/main.py
import httpx, os
from fastapi import FastAPI, Request, HTTPException, Depends
from fastapi.middleware.cors import CORSMiddleware
from shared.auth import decode_token

app = FastAPI(title="API Gateway")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],   # tighten in production
    allow_methods=["*"],
    allow_headers=["*"],
)

ROUTES = {
    "/users":    os.getenv("USER_SERVICE_URL"),
    "/products": os.getenv("PRODUCT_SERVICE_URL"),
    "/orders":   os.getenv("ORDER_SERVICE_URL"),
}

# Routes that don't require authentication
PUBLIC_PATHS = {"/users/register", "/users/login", "/products"}

async def verify_auth(request: Request) -> dict | None:
    path = request.url.path
    if any(path.startswith(p) for p in PUBLIC_PATHS):
        return None
    auth = request.headers.get("Authorization", "")
    if not auth.startswith("Bearer "):
        raise HTTPException(401, "Missing token")
    return decode_token(auth.split()[1])

Request Routing

A single catch-all route proxy that forwards any request to the correct service, injects the verified user ID, and streams back the response.

Universal Proxy Handler

api-gateway/main.py (continued)
from fastapi.responses import Response @app.api_route("/{path:path}", methods=["GET","POST","PUT","DELETE","PATCH"]) async def proxy(path: str, request: Request, claims=Depends(verify_auth)): # Find the right service for this path prefix target_url = None for prefix, base_url in ROUTES.items(): if request.url.path.startswith(prefix): target_url = base_url + request.url.path break if not target_url: raise HTTPException(404, "Unknown service") # Forward headers + inject user context headers = dict(request.headers) if claims: headers["X-User-ID"] = str(claims["user_id"]) headers["X-User-Email"] = claims["sub"] async with httpx.AsyncClient() as client: resp = await client.request( method=request.method, url=target_url, headers=headers, content=await request.body(), params=request.query_params, timeout=30.0 ) return Response( content=resp.content, status_code=resp.status_code, headers=dict(resp.headers), media_type=resp.headers.get("content-type") )
Note
Services receive X-User-ID and X-User-Email headers. They trust these because they're in a private Docker network — not reachable from the public internet. Never expose service ports externally.

Rate Limiting & CORS

Protect your services from abuse by limiting request rates with Redis counters, and configure CORS properly for browser clients.

Redis-backed Rate Limiter

api-gateway/middleware.py
import redis.asyncio as redis
from fastapi import Request, HTTPException
import os, time

pool = redis.ConnectionPool.from_url(os.getenv("REDIS_URL", "redis://redis:6379"))

async def rate_limit(request: Request, limit: int = 60, window: int = 60):
    # Identify by IP (or authenticated user ID)
    client_ip = request.client.host
    key = f"rate:{client_ip}:{int(time.time()) // window}"
    
    r = redis.Redis(connection_pool=pool)
    count = await r.incr(key)
    
    if count == 1:
        await r.expire(key, window)   # set TTL on first hit
    
    if count > limit:
        raise HTTPException(
            status_code=429,
            detail="Too many requests",
            headers={"Retry-After": str(window)}
        )

# Apply as a dependency on the proxy route
@app.api_route("/{path:path}", methods=["GET","POST","PUT","DELETE"])
async def proxy(request: Request, _=Depends(rate_limit), claims=Depends(verify_auth)):
    ...

Chapter 12 · Service Communication

Synchronous HTTP (httpx)

When a service needs data from another service immediately — like Order Service checking product price — it makes a direct HTTP call using httpx.

Async HTTP Client with Retry

order-service/http_client.py
import httpx
from tenacity import retry, stop_after_attempt, wait_exponential

# Reuse a single client (connection pooling)
_client: httpx.AsyncClient | None = None

async def get_client() -> httpx.AsyncClient:
    global _client
    if _client is None:
        _client = httpx.AsyncClient(timeout=10.0)
    return _client

@retry(stop=stop_after_attempt(3), wait=wait_exponential(min=0.5, max=4))
async def get_product(product_id: int) -> dict:
    client = await get_client()
    resp = await client.get(f"{PRODUCT_URL}/products/{product_id}")
    resp.raise_for_status()  # raises on 4xx/5xx
    return resp.json()

# Startup/shutdown lifecycle
@app.on_event("startup")
async def startup():
    await get_client()

@app.on_event("shutdown")
async def shutdown():
    if _client: await _client.aclose()
Tip
Always use a shared client (connection pool) rather than creating a new httpx.AsyncClient() per request. The latter opens a new TCP connection every time — catastrophic under load.

Async Messaging with RabbitMQ

For fire-and-forget operations — notifications, analytics writes, audit logs — services publish events to a message broker. Consumers process them independently and at their own pace.

Exchange Types

TypeRoutingUse case
directExact routing keyTask queues, 1-to-1
topicPattern (order.*)Event bus, pub/sub
fanoutAll queuesBroadcast, logging

Publisher (Order Service)

order-service/publisher.py
import aio_pika, json

class EventPublisher:
    def __init__(self, rabbitmq_url: str):
        self.url = rabbitmq_url
        self.connection = None

    async def connect(self):
        self.connection = await aio_pika.connect_robust(self.url)
        self.channel = await self.connection.channel()
        self.exchange = await self.channel.declare_exchange(
            "orders", aio_pika.ExchangeType.TOPIC, durable=True
        )

    async def publish(self, routing_key: str, payload: dict):
        await self.exchange.publish(
            aio_pika.Message(
                body=json.dumps(payload).encode(),
                delivery_mode=aio_pika.DeliveryMode.PERSISTENT  # survive restart
            ),
            routing_key=routing_key
        )

publisher = EventPublisher(os.getenv("RABBITMQ_URL"))

Chapter 14 · Service Communication

Event-Driven Patterns

Standardize your events with a consistent schema, implement idempotency to handle duplicate deliveries, and use dead-letter queues for failed messages.

Standard Event Schema

shared/events.py
from pydantic import BaseModel, Field
from datetime import datetime
from uuid import uuid4

class BaseEvent(BaseModel):
    event_id: str = Field(default_factory=lambda: str(uuid4()))
    event_type: str
    source_service: str
    timestamp: datetime = Field(default_factory=datetime.utcnow)
    payload: dict

# Usage
event = BaseEvent(
    event_type="order.created",
    source_service="order-service",
    payload={"order_id": 42, "user_id": 7}
)

Idempotency — Handle Duplicate Messages

notification-service/consumer.py
async def handle_order_created(message: aio_pika.IncomingMessage):
    async with message.process():
        event = BaseEvent(**json.loads(message.body))
        
        # Redis dedup: skip if we've already processed this event_id
        already_processed = await redis.get(f"processed:{event.event_id}")
        if already_processed:
            return  # idempotent — safe to skip
        
        # Process the event
        send_email(event.payload)
        
        # Mark as processed (expire after 24h)
        await redis.setex(f"processed:{event.event_id}", 86400, "1")
Critical
RabbitMQ guarantees at-least-once delivery — your consumers must be idempotent. The same message can arrive twice after a network hiccup. Always deduplicate using the event_id.

Health Checks

Every service exposes a /health endpoint so Docker, Kubernetes, and the gateway can verify it's alive and ready to serve traffic.

Comprehensive Health Endpoint

shared/health.py
from fastapi import FastAPI
from sqlalchemy.orm import Session
from database import SessionLocal
import time

def add_health_routes(app: FastAPI):
    
    @app.get("/health", tags=["Health"])
    async def health():
        checks = {}
        
        # DB check
        try:
            db: Session = SessionLocal()
            db.execute("SELECT 1")
            checks["database"] = "ok"
        except Exception as e:
            checks["database"] = "error"
        finally:
            db.close()

        status = "healthy" if all(v == "ok" for v in checks.values()) else "degraded"
        return {
            "status": status,
            "checks": checks,
            "timestamp": time.time()
        }
    
    @app.get("/ready", tags=["Health"])
    async def ready():
        # Called by k8s readiness probe
        return {"ready": True}

Chapter 16 · Resilience & Ops

Circuit Breaker

Prevent cascading failures. When Product Service is down, Order Service shouldn't keep hammering it — it should fail fast and return a cached or degraded response.

Circuit Breaker with pybreaker

order-service/http_client.py
import pybreaker
import httpx

# Open circuit after 5 failures; retry after 30s
product_breaker = pybreaker.CircuitBreaker(
    fail_max=5,
    reset_timeout=30
)

@product_breaker
async def get_product_price(product_id: int) -> float:
    async with httpx.AsyncClient() as c:
        resp = await c.get(f"{PRODUCT_URL}/products/{product_id}")

async def safe_get_product_price(product_id: int) -> float | None:
    try:
        return await get_product_price(product_id)
    except pybreaker.CircuitBreakerError:
        # Circuit is OPEN — fail fast, return cached or None
        cached = await redis.get(f"product_price:{product_id}")

Distributed Tracing

Correlate logs and trace a single request as it hops through multiple services using a shared X-Request-ID header and structured logging.

Request ID Propagation

shared/tracing.py
from fastapi import Request
from starlette.middleware.base import BaseHTTPMiddleware
import uuid, logging, time, json

logger = logging.getLogger("service")

class TracingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        request_id = request.headers.get("X-Request-ID", str(uuid4()))
        start = time.time()
        
        response = await call_next(request)
        
        duration_ms = (time.time() - start) * 1000
        logger.info(json.dumps({
            "request_id": request_id,
            "method": request.method,
            "path": request.url.path,
            "status": response.status_code,
            "duration_ms": round(duration_ms, 2)
        }))
        
        response.headers["X-Request-ID"] = request_id
        return response

# Add to every service
app.add_middleware(TracingMiddleware)

Redis Caching

Cache expensive reads — product listings, user profiles — in Redis to reduce database load and dramatically improve response times.

Cache Decorator

shared/cache.py
import redis.asyncio as aioredis
import json, functools
from typing import Callable

redis_client = aioredis.from_url("redis://redis:6379")

def cache(ttl: int = 300, prefix: str = ""):
    """Decorator: cache the result in Redis for `ttl` seconds."""
    def decorator(func: Callable):
        @functools.wraps(func)
        async def wrapper(*args, **kwargs):
            key = f"{prefix}:{func.__name__}:{args}:{kwargs}"
            cached = await redis_client.get(key)
            if cached:
                return json.loads(cached)       # cache HIT
            result = await func(*args, **kwargs)
            await redis_client.setex(key, ttl, json.dumps(result))
            return result                        # cache MISS → stored
        return wrapper
    return decorator

# Usage in product-service
@cache(ttl=120, prefix="products")
async def get_all_products(skip: int, limit: int):
    return ... # DB query

# Invalidate on product update
async def invalidate_product_cache(product_id: int):
    pattern = f"products:*"
    keys = await redis_client.keys(pattern)
    if keys:
        await redis_client.delete(*keys)

JWT Across Services

A unified authentication strategy: the User Service issues JWTs, the Gateway verifies them, and services trust the injected headers — eliminating redundant token checks.

The Auth Flow End-to-End

1
POST /users/login → User Service
Returns {"access_token": "eyJ...", "token_type": "bearer"}
2
Client stores token, sends on every request
Authorization: Bearer eyJ... header on all protected calls
3
Gateway verifies the JWT
Calls decode_token() from shared lib. Returns 401 if invalid/expired.
4
Gateway injects X-User-ID header
Services read X-User-ID via the get_user_id dependency — no re-verification needed.
